Competenties

Waarom kies je voor de opleiding programmeren?

De reden dat ik voor het beroep van programmeur heb gekozen, is dat ik van programmeren houd. Een programmeur is een zeker een creatief persoon. Snel oplossingen vinden voor bepaalde problemen, ze succesvol oplossen, fouten in de code vinden en elimineren, de code vereenvoudigen en verbeteren - dit is wat ik zal moeten doen. Gemotiveerde programmeurs schrijven de meest aantrekkelijke en gebruikvriendelijke programma's voor gebruikers. Tegenwoordig is er altijd een baan voor een programmeur, er is vraag naar hem in bijna alle sferen van het leven, zijn werk wordt zeer gewaardeerd en de vraag naar zijn activiteiten groeit in de loop van tijd. Een goede programmeur kan altijd rekenen op een fatsoenlijk salaris. Door de grote vraag naar deze specialisten is de concurrentie tussen werkgevers vrij hoog. Een programmeur is één van de weinige beroepen die de constante ontwikkeling van bepaalde vaardigheden omvat. Veel producten die programmeur bij het maken betrokken is, brengen grote winsten met zich mee: ze worden in grote hoeveelheden verkocht, of ze zijn duur, of beide.

Waarom denk je dit beroep bij jou past?

Ik vind het leuk om betrokken te zijn bij het creëren van een project en samen te werken met andere mensen. Ik ban van mening dat goed werk altijd goed beloond wordt en er een stimulans is om de kwaliteit van de code te verbeteren. Ik vind het leuk dat het mogelijk is om overal ter wereld te werken waar een internetverbinding is. De mogelijkheid van permanente opleiding is ook erg aantrekkelijk voor mij. Ik wil verder studeren om nieuwe technologieën, features en programmeertalen te leren.

Competenties programmeur

Ontwerper

Waar sta ik momenteel?

De essentie van het onderwerp is duidelijk. De taken worden uitgevoerd op
een bovengemiddeld niveau. Het is duidelijk hoe HTML5 en CSS werken en hoe je een website maakt. Waar te zoeken naar een oplossing voor het probleem, hoe het op te lossen. In principe is er een gevoel dat dit deel van programmering onder de knie is. Er is niet genoeg werkervaring.

Wat lukt al goed?

De basis van HTML en CSS is all duidelijk. Het is al mogelijk om de website te visualiseren met behulp van HTML en CSS-technologien. Over het algemeen ken ik de semantic HTML5, CSS Layout, Flexbox. Hoe maak je een tabel en achtergrond, hover en nth-child elementen en zo verder.

Wat lukt nog niet?

Niet alle functies van het onderwerp zijn nog beschikbaar. Veel dingen ( zoals tags of sintax) leukt alleen met behulp van internet en Google vinden. Ik heb een vast gevoel dat er nog veel te leren valt op het gebied van frontend. Maar nog steeds veel dingen worden intuïtief verkregen op dit moment. Ik kan me nog niet realiseren hoe het werkt, maar het werkt.

Voorbeelden

Ik heb alle taken van de Web Essentials-cursus allen voltooid, maar met de zorgvuldige begeleiding van onze docent. Ik ben erin geslaagd om een tabellen te bouwen en de Sony Playstation-website praktisch na te maken. Ik heb geleerd hoe ik mijn eigen portfolio en curriculum vitae kan maken op zo'e krachtig platform als Web Storm . We weten al hoe we de user input tool moeten programmeren.

Programmeur

Waar sta ik momenteel?

Aan de begin van een lang maar interessant ontwikkelingstraject als softwareontwikkelaar. Hier is natuurlijk alles veel gecompliceerder dan in webontwikkeling , er zijn logica en een compieler die heel erg de syntax bewaakt. We moeten meer inspanningen leveren om deze muur van onbegrip te doorbreken. Maar ik vind het leuk , want met begrip komt ook liefde voor deze programmeertaal. Terwijl ik C# onder de knie heb, vond ik het steeds leuker !

Wat lukt al goed?

Het is al mogelijk om zelf code te schrijven in C# en een grafische interface te bouwen. Soms luekt het om een programma helemaal zelf te schrijven. Het blijkt variabelen toe te wijzen, methoden te definiëren en ze aan elkaar te koppelen, loops en arrays te maken. XML code schrijven is tot nu toe de gemakkelijkste wat ik kan creëren in .NET.

Wat lukt nog niet?

Het is niet direct mogelijk om snel nieuwe methoden onder de knie te krijgen. "Herhaling is de moeder van het leren" - alleen door voortdurend te herhalen kun je deze taal beheersen. Soms lijkt het me dat programmeren magie is, omdat niet duidelijk is hoe deze code kan werken, maar het werkt, en vice versa. Het meest onaangename is wanneer je een fout lange tijd niet kan oplossen , dan begin je onbewust allerlei methoden op te pikken en weg te gaan van de essentie van het probleem.

Voorbeelden

Hier, zoals ze zeggen, is er nog een groot onbewerkt veld. Nog steeds werken en werken. Ik besteed veel tijd aan het bestuderen van C#, Ik studeer theorie uit een boek "Programmeren in C#", lees de cursus en bekijk video's, los taken uit het programma op en natuurlijk CookieClicker. Terwijl het op de een of andere manier niet perfect blijkt. Het kan altijd beter zijn.

Tester

Waar sta ik momenteel?

Tot nu toe kan ik allen da geschrijven code visueel testen. Ik begrijp dat een tester niet alleen zijn persoonlijke code moet kunnen begrijpen , maar ook de code van andere programmeurs. Ik wil eerst leren hoe ik zelf schone en perfecte code kan schrijven. Ik vind het leuk om de mooie code van iemand anders te begrijpen, te beseffen hoe het werkt, nieuwe methoden ten leren, schoon en mooi te leren schrijven.

Wat lukt al goed?

De code valideren, visueel testen. Het belangrijkste is dat de code correct werkt. Ik test mijn code na elke implementatie. Het vinden van je eigen bugs, is routinewerk van elke programmeur. En dit is een van de eenvoudigste taken, de moeilijkste om bugs op te lossen. Maar voor zover ik weet repareren testers geen bugs, maar zoeken ze alleen.

Wat lukt nog niet?

Ik ken nog geen testprogrammas en testmethoden. Ik weet dat er speciale programma's zijn voor het testen van de programmacode. Er zijn ook ingebouwde hulpprogramma's in de Visual Studio zelf, maar ik heb nog niet geleerd hoe ik ze moet gebruiken.

Voorbeelden

Communicator/Teamspeler

Waar sta ik momenteel?

Op dit moment zie ik mezelf niet als projectleider of teamleider. Ik wil eerst een leider voor de computer worden, dit is mijn primary doel. Daarom concentreer ik me meer op mijn wiskundige vaardigheiden en in het oplossen van ingestelde taken. Meer geïnteresseerd in het leren van verschillende o oplossingsmethoden en technologien in programmeren. De rol van een mentor wordt interessanter naarmate kennis en ervaring toenemen.

Wat lukt al goed?

Presentaties zijn niet slecht. Het is belangrijkste is om de klant goed te begrijpen. Voordat je op een project concentreerd , moet u zich concentreren op de verzoek van de klant. Een productpresentatie kan u helpen op raakvlakken te vinden met de klant of om het begrip te testen.

Wat lukt nog niet?

Nederlands is niet mijn moedertaal. Aan ene kant kan ik me meer richten op het begrijpen van de klant. Hierdoor kunnen we sneller overshakelen naar de visualisatietaal. Aan de andere kant geven de grote klanten van grote projecten in de zakelijke omgeving er de voorkeur aan om vloeiend te communiceren in hun moedertaal.
